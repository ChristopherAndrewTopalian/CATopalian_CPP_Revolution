# Tutorial: Your First C++ Program (Print and Pause)

Welcome to the CATopalian C++ Revolution.

If you have ever written a script before (like JavaScript, Python, or LSL), you know that a script is just a list of instructions telling the computer what to do. C++ is exactly the same, but it is built for extreme speed and security.

In this first lesson, we are going to write a simple program that prints a message to the screen and then waits patiently for you to press Enter before closing.

### The Code

```cpp
// print_and_keep_open

#include <iostream>

int main()
{
    std::cout << "Hi Everyone";

    // Standardized Exit Pattern
    std::cout << "\nPress Enter to Exit";
    std::cin.ignore();
    std::cin.get();
    
    return 0;
}

```

### Breaking It Down (Line by Line)

To a beginner, some of these words might look strange. Let's translate exactly what the engine is doing under the hood.

* **`#include <iostream>`**
Think of this as importing a module or a plugin. `iostream` stands for **Input/Output Stream**. By including this at the very top, we are giving our script the ability to print words to the monitor (Output) and listen to the keyboard (Input).
* **`int main()`**
This is the Engine Room. Every single C++ program on Earth must have a `main()` function. When you double-click your program to run it, the computer immediately looks for `main()` and starts executing the code inside its brackets `{ }`.
* **`std::cout << "Hi Everyone";`**
* `std::` stands for the **Standard Library**. It tells the compiler we are using a safe, built-in tool.
* `cout` stands for **Character Output**.
* The two arrows `<<` act like a conveyor belt, pushing the text `"Hi Everyone"` directly onto the screen.
* *Note: Just like in many scripting languages, every instruction in C++ must end with a semicolon (`;`).*


* **`\n`**
This is a special invisible character that means **"New Line."** It acts exactly like hitting the Enter key on your keyboard, pushing the next line of text down so things don't get squished together.

#### The Standardized Exit Pattern

If we just printed the text and finished the script, the computer would execute it in a millisecond, and the black terminal window would flash and close before you could even read the words! We use this block of code to force the window to stay open.

* **`std::cin.ignore();` and `std::cin.get();**`
* `cin` stands for **Character Input** (listening to the keyboard).
* These two commands act as a security gate. They force the program to completely freeze and wait for the user to physically press the `Enter` key on their keyboard before moving forward.


* **`return 0;`**
This is how our program salutes the Operating System on its way out. Returning `0` is the universal signal for: *"Mission accomplished, the program ran perfectly with zero errors."*

### How to Build and Run Your Program

Unlike JavaScript, which runs inside a web browser, C++ code must be **compiled**. This means we use a tool (the compiler) to translate our human-readable text into a lightning-fast, standalone `.exe` application.

To compile this script, open your terminal and type this exact command:

`g++ main.cpp -std=c++20 -o our_test.exe`

* `g++`: The name of our compiler.
* `main.cpp`: The text file containing our code.
* `-std=c++20`: This is crucial! It tells the compiler to use the modern **2020 C++ Standard**, unlocking the cleanest and most secure features of the language.
* `-o our_test.exe`: This tells the compiler to output a finished, runnable program named `our_test.exe`.

### The Output

Once compiled, run `our_test.exe`. You will see this perfectly formatted output:

```text
Hi Everyone
Press Enter to Exit

```
